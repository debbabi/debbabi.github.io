TP Langages et Compilation > Passe 3


1. Introduction

L'objectif est de réaliser UNE passe 3 par groupe d'étudiants (soit 8 à 9 étudiants par groupe). Une des équipes (binôme) fournit les passes 1 et 2.

Le but de la passe 3 est de générer du code pour une machine abstraite (cf. MachineAbstraite.txt).

On a un programme JCas, contextuellement correct, et on cherche à produire un fichier assembleur qui contient le code correspondant à ce programme.

La passe 3 peut ne pas réussir à générer du code ; dans ce cas la compilation doit être interrompue et un message d'erreur affiché, avec le numéro de ligne de l'erreur dans le programme source.

Classes fournies (Src3.tar.gz, Gencode.tar.gz)

La classe principale du compilateur JCas est JCasc ("JCas compiler"), dans le paquetage ProjetCompil.Gencode.Src. Dans le répertoire ProjetCompil/Gencode/Test, la commande jcasc fich.cas produit, si la compilation réussit, un fichier assembleur fich.ass.

Ce fichier peut être exécuté par l'interprète de machine abstraite : ima fich.ass.

Le paquetage ProjetCompil.Global.Src3 fournit des primitives pour manipuler le programmes assembleur généré. Le programme est une suite de lignes. Une ligne comporte une étiquette, une instruction et un commentaire (chacun de ces éléments étant optionnel). Une instruction comporte une opération est 0, 1 ou 2 opérandes. (cf. classes Prog, Ligne, Inst, Etiq, Operation, Operande, NatureOperande).

3. Planning détaillé

  • Séance 7 : Préparation de l'environnement de travail
    1. Constitution des équipes
    2. Création d'un dépot svn centralisé (pour le partage du code source entre les membres d'équipe)
    3. Installation de la machine abstraite (chaque binôme)
    4. Lecture de la documentation: MachineAbstraite.txt, Ima.txt, cours de compilation, svn, ...

  • Séance 8 : Architecture
    • Création des interfaces : classes et méthodes publiques de chaque classe. Chaque méthode est créée avec un corps vide et chaque classe est placée dans le projet.
    • Choix des algorithmes de génération de code (voir le cours de compilation)
    • Affectation des tâches pour chaque binôme
    • Rédaction des pages de documentation
      • Tâches / Binômes
      • Planning : diagramme de gantt
      • L'architecture du projet : diagramme de classes, séquence, ...
    • Début de développement du code

  • Séance 9 : Développement
    • Développement du code
    • Premiers tests
    • Debug

  • Séance 10 : Tests
    • Fin de développement
    • Ecriture de tests plus complexes
    • Debug

  • Séance 11 : Fin du projet
    • Ecriture d'une batterie complète de tests
    • Objectif : obtenir un compilateur zéro-défaut

4. Rendu de la passe 3

  • S'assurer que toutes les modifications de tous le binômes ont été mis à jours dans la source globale.
  • L'enseignant effectuera un checkout du source à la date indiquée dans le planning. Aucune autre modification effectuée après cette date ne sera prise en compte.
