Consignes

Voici quelques consignes à respecter lors du codage de vos exercices. Ils sont prises en considération lors de l'évaluation de vos solutions.

  1. Utilisation des constantes dès que possible. Les identificateurs des constantes doivent être composés de MAJUSCULES
  2. Indentation dans le code source à respecter
  3. Traitement des erreurs
  4. Commentaires et documentation du code source :
    • L'en-tête de chaque fichier, composée d'une description globale du code et les noms des développeurs auteurs du code;
    • commentaire des fonctions : paramètres acceptés et valeurs de retour possibles;
    • commentaire du détail de programmation (variables, boucle, etc.)
    • Exemple :
/* Description globale du code.  
    Etudiants : 
    - Nom1
    - Nom2
*/

#define MAX  10                             /* préviligiez l'utilisation des constantes */

 * v : une valeur
 * retourne : -1 en cas d'erreur, 0 dans les autres cas
 */

{

                                              si nécessaire */
